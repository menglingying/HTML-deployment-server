# 阿里云部署说明（多设备数据同步）

## 1. 本项目现在的结构

- `index.html`：前端页面（已接入 `/api/state` 云端同步）
- `没命了.html`：原始页面备份（内容与 `index.html` 同步）
- `server.js`：Node.js 后端（统一存储数据，支持版本冲突检测）
- `package.json`：依赖和启动命令
- `ecosystem.config.cjs`：PM2 进程守护配置

## 2. 先决条件（阿里云）

- 你需要一台 ECS（Ubuntu 24.04 可用）。
- 安全组入方向放通：
  - `22`（SSH）
  - `80`（HTTP）
  - `443`（HTTPS）
- 你需要一个域名，并在中国大陆合规上线时完成 ICP 备案。

## 3. 上传代码到服务器

在你本地 PowerShell 执行（把 `<服务器IP>` 换成你的 ECS 公网 IP）：

```powershell
Compress-Archive -Path `
  "E:\html部署服务器\index.html",`
  "E:\html部署服务器\没命了.html",`
  "E:\html部署服务器\logo.png",`
  "E:\html部署服务器\card-swipe.jpg",`
  "E:\html部署服务器\server.js",`
  "E:\html部署服务器\package.json",`
  "E:\html部署服务器\package-lock.json",`
  "E:\html部署服务器\ecosystem.config.cjs" `
  -DestinationPath "E:\html部署服务器\orange-heart.zip" -Force

scp "E:\html部署服务器\orange-heart.zip" root@<服务器IP>:/opt/orange-heart/
```

服务器上执行：

```bash
mkdir -p /opt/orange-heart
cd /opt/orange-heart
unzip -o orange-heart.zip
ls
```

## 4. 安装 Node.js 与 PM2（服务器上）

```bash
apt update
apt install -y curl ca-certificates gnupg
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs nginx unzip
npm install -g pm2
```

确认版本：

```bash
node -v
npm -v
pm2 -v
```

## 5. 安装项目依赖并启动

```bash
cd /opt/orange-heart
npm install
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup systemd -u root --hp /root
```

健康检查：

```bash
curl http://127.0.0.1:3000/api/health
```

## 6. 配置 Nginx 反向代理

创建配置文件：

```bash
cat > /etc/nginx/sites-available/orange-heart.conf <<'EOF'
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF
```

启用并重载：

```bash
ln -sf /etc/nginx/sites-available/orange-heart.conf /etc/nginx/sites-enabled/orange-heart.conf
nginx -t
systemctl reload nginx
```

## 7. 阿里云域名解析（A 记录）

在阿里云域名控制台添加：

- 主机记录：`@`，记录类型：`A`，记录值：`<服务器公网IP>`
- 主机记录：`www`，记录类型：`A`，记录值：`<服务器公网IP>`

等待解析生效后，访问：

- `http://your-domain.com`
- `http://www.your-domain.com`

## 8. 配置 HTTPS（推荐）

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d your-domain.com -d www.your-domain.com
systemctl status certbot.timer
```

## 9. 多设备同步验证

1. 设备 A 打开域名，登录管理员，新增/修改学员并保存。
2. 设备 B 打开同一域名，等待几秒（系统每 8 秒拉取云端更新）。
3. 设备 B 应看到与设备 A 一致的数据。

## 10. 已实现的同步机制

- 前端每次改动会：
  - 先保存本地缓存（防断网）
  - 再提交到 `/api/state`
- 前端每 8 秒自动拉取云端新版本
- 后端有 `version` 字段，冲突时返回 `409`，前端会自动回拉最新数据

## 11. 运维常用命令

```bash
pm2 status
pm2 logs orange-heart-points --lines 100
pm2 restart orange-heart-points
systemctl status nginx
tail -n 100 /var/log/nginx/error.log
```

## 12. 数据文件位置

- 后端统一存储：`/opt/orange-heart/data/state.json`
- 建议定时备份该文件（例如每天一次）。
